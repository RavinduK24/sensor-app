7. HOW IT WORKS
----------------
Real-Time Data Generation:
  1. Sensor simulator generates readings at different rates
     - SOUND: 1 second
     - AIR_QUALITY: 10 seconds
     - TEMPERATURE/HUMIDITY/LIGHT: 10 minutes
  
  2. All readings stored in RealtimeReading table

10-Minute Interval Aggregation:
  1. API endpoint divides 24 hours into 144 intervals
  2. For each 10-minute window:
     - Query all readings in that window
     - Calculate average value
     - Return as data point
  3. Frontend plots all 144 points on 0-23 hour scale

Display:
  - X-axis shows hours (0, 2, 4, ..., 22, 24)
  - Data points plotted at decimal positions
  - Smooth line connects all 144 points
  - Tooltip shows precise time (HH:MM)
